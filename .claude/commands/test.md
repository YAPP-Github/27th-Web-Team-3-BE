# /test - 테스트 실행 명령어

테스트를 실행하고 결과를 분석합니다.

## 실행 단계

1. `cargo test` - 모든 테스트 실행
2. 실패한 테스트 분석
3. 필요시 수정 제안

## 옵션

- `/test` - 모든 테스트 실행
- `/test [name]` - 특정 테스트만 실행
- `/test --nocapture` - 출력 포함하여 실행

## 실패 시 행동

1. 실패한 테스트 식별
2. 관련 코드 분석
3. TDD 원칙에 따라 수정안 제시
4. 사용자 승인 후 수정

## 출력 형식

```
테스트 결과: X passed, Y failed
실패한 테스트:
- test_name: 실패 원인
```
